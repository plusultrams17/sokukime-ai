## 1. 営業系TikTok/Reelsのバズ分析

### 1-1. 2026年TikTokアルゴリズムの決定的変化

2026年のTikTokは「検索エンジン・生活インフラ」化が加速。かつての「瞬発力のバズ勝負」から「持続力・信頼構築の勝負」へ構造変化が起きている。

#### アルゴリズム最重要指標（2026年4月時点）

| 指標 | 目標値 | 重要度 | 備考 |
|------|--------|--------|------|
| 視聴完了率（15秒以下） | 50%以上 | 最高 | バズのボーダーは70%に上昇 |
| 視聴完了率（30秒以上） | 30%以上 | 最高 | |
| 保存率 | 1%以上 | 極高 | 2026年最重視指標 |
| シェア率 | 0.5%以上 | 高 | DMシェアがアルゴリズム優遇 |
| いいね率 | 3%以上 | 中 | |
| コメント率 | 0.1%以上 | 中 | |
| プロフィール遷移率 | 1%以上 | 中 | |

**2026年の重要変化:**
- 5秒以上の視聴が「クオリファイド・ビュー（有効な視聴）」としてカウント
- 「率」が「合計数」より重視される（フォロワー0でもバズ可能）
- アカウントレベルの信頼スコアが導入（投稿の一貫性・ジャンル特化が評価）
- 検索SEO対応: テロップ・字幕・音声内のキーワードがアルゴリズムに影響

### 1-2. バズっている営業・ビジネス系フォーマット

#### フォーマットA: 「一言パワーフレーズ」型（最も再現しやすい）

```
フック（0-3秒）: テロップで衝撃フレーズ
  例: 「『検討します』と言われたら、この一言で逆転できる」
展開（3-12秒）: 具体的なトーク例を引用符表示
  例: 「『ありがとうございます。ちなみに、何を検討されますか？』」
オチ（12-15秒）: 数字で裏付け + CTA
  例: 「この切り返しだけで成約率が14%→31%に。練習はプロフのリンクから」

推奨尺: 15秒以内
BGM: 落ち着いたビジネス系BGM or トレンド音源
```

**なぜ伸びるか:**
- 「すぐ使える」実用性 → 保存率が高い
- 短尺で視聴完了率が稼ぎやすい
- 「同僚に教えたい」→ DMシェア誘発

#### フォーマットB: 「Before/After成果報告」型

```
フック（0-3秒）: 数字の変化をドン！と表示
  例: 「成約率 8% → 31%」テロップ + 驚き顔
展開（3-20秒）: 何をやったか3ステップで説明
  例: Step1: AIでロープレ毎日10分 / Step2: フィードバックで弱点特定 / Step3: 切り返し集中練習
オチ（20-30秒）: 画面録画でAIロープレのデモを見せる

推奨尺: 30秒
BGM: アップテンポな達成感系
```

**なぜ伸びるか:**
- 数字の変化が視覚的にインパクト大
- プロダクト導線が自然に組み込める
- 「自分もやってみたい」→ プロフィール遷移

#### フォーマットC: 「あるあるネタ×営業」型

```
フック（0-2秒）: 「営業マンなら100%共感するやつ」テロップ
展開（2-10秒）: シチュエーション再現（一人二役 or テロップ劇）
  例: 「『検討します』が出た瞬間の営業マンの脳内」
オチ（10-15秒）: 「でも練習してたら焦らなくなった」+ CTA

推奨尺: 15秒
BGM: コメディ系 or ポップ系トレンド音源
```

**なぜ伸びるか:**
- 共感でコメント爆発 → アルゴリズム75xボーナス相当
- 「あの同僚に送りたい」→ DMシェア
- 軽いノリなのでTikTok文化と相性が良い

#### フォーマットD: 「AIデモ実況」型（差別化最強）

```
フック（0-3秒）: 「AIに商談の練習相手になってもらった結果...」
展開（3-40秒）: 画面録画でAIロープレの実際のやり取りを見せる
  - ユーザーの発言 → AIの反応 → フィードバック表示
オチ（40-60秒）: スコア結果を見せて「毎日これやってたら上司に褒められた」

推奨尺: 45-90秒（SaaSデモは長尺が許容される傾向）
BGM: Lo-Fi系 or なし（音声重視）
```

**なぜ伸びるか:**
- 「AI × 営業」の組み合わせの新規性
- 日本のTikTokで営業AIロープレ動画はほぼ皆無 → ブルーオーシャン
- SaaSデモ動画は45-90秒の長尺でも視聴完了される傾向（2026年データ）

## 2. X (Twitter) の営業系コンテンツ分析

### 2-1. 2026年Xアルゴリズム（Phoenix）の仕組み

2026年1月に導入された新アルゴリズム「Phoenix」の特徴:

#### スコアリング倍率（2026年4月確認）

| アクション | スコア | 戦略的意味 |
|-----------|--------|-----------|
| 投稿者からのリプライ返信 | 75.0pt | 最高評価。リプにはすべて返信すべき |
| リプライ | 13.5pt | 会話を生む投稿が正義 |
| 詳細クリック後のいいね/リポスト | 11.0pt | 長文を読了させる設計が有効 |
| リポスト | 1.0pt | 拡散だが倍率は低い |
| いいね | 0.5pt | ベースライン |
| 画像/動画付き | 2x | テキストのみ比 |
| Xプレミアム会員 | 2-4x | フォロワーには4倍 |
| 外部リンク直貼り | -30~-50% | ペナルティ。リプライに分離 |
| ミュート/ブロック | -500pt | 致命的 |
| 通報 | -1,500pt | 致命的 |

#### 表示優先度の計算式（推定）

```
関連性スコア (30%)
+ エンゲージメント予測値 (25%)
+ コンテンツ新鮮度 (20%)
+ ユーザー信頼度 (15%)
+ 多様性要素 (10%)
= タイムライン表示優先度
```

**最重要ポイント:**
- リプライの往復は「いいね」の150倍の価値
- 投稿を2分以上読まれると22xの評価
- 1日5投稿より質の高い1投稿の方が伸びやすい
- 投稿後15-30分以内のリプライ返信がバズの分岐点

## 3. AI x SaaS マーケティング成功事例

### 3-1. 日本のAI SaaS SNS成功事例

#### 事例A: Notion（海外→日本進出）
- **手法**: TikTokでZ世代学生のUGC（ユーザー生成コンテンツ）を促進
- **成功要因**: 製品自体がビジュアル映え → 「私のNotionセットアップ」系動画が自然発生
- **学び**: プロダクト画面そのものをコンテンツにする戦略。成約コーチAIでも「AIロープレ画面」をそのまま見せる動画が有効

#### 事例B: 入江慎吾氏（日本 BuildInPublic先駆者）
- **手法**: Xで開発過程をリアルタイム発信
- **成功要因**: MRR、ユーザー数、売上の数字公開が最も拡散
- **学び**: 「使ってみてください」より「こんな機能作りました、どう思いますか？」の問いかけ型が有効

#### 事例C: SABON（SNS広告 x UGC）
- **手法**: インフルエンサーによる開封動画をSpark Adsで広告配信
- **成果**: ギフトセット売上 前年比140%
- **学び**: UGC + 広告の組み合わせ。成約コーチAIでは「AIロープレ体験動画」をユーザーに作ってもらい、それを広告配信する導線

### 3-2. 海外AI SaaSマーケティング参考事例

#### 事例D: Jasper AI
- **手法**: デモ動画で「AIがコピーを書く瞬間」をそのまま見せる
- **成功要因**: 「魔法のような瞬間」をビジュアルで体感させる
- **学び**: 成約コーチAIでは「AIが商談フィードバックを返す瞬間」を見せる動画が同等のインパクトを持つ

#### 事例E: SaaS企業のデモ動画トレンド
- **2026年の変化**: TikTokでSaaSデモ動画は45-90秒の長尺が許容される
- **成功条件**: 「研究モード中のユーザー」にアルゴリズムが長尺コンテンツを配信
- **学び**: 商談練習に興味があるユーザーには、しっかり見せるデモが刺さる

### 3-3. プロダクトをコンテンツにする方法

成約コーチAIで実行すべき「プロダクト=コンテンツ」戦略:

| コンテンツ種類 | 具体的な作り方 | 期待効果 |
|--------------|--------------|---------|
| AIロープレ画面録画 | 実際にAIと商談練習する画面を30-60秒で録画 | 新規性+デモ+信頼 |
| スコア結果Before/After | 初回と30日後のスコア画面を並べて表示 | 成果の可視化 |
| フィードバック切り抜き | AIが返した秀逸なフィードバックをスクショ | 「これが月990円？」の驚き |
| 失敗ロープレ→改善ロープレ | NG例とOK例を対比する動画 | 教育コンテンツとして保存される |
| ユーザーの実体験動画 | ユーザーに体験動画を撮ってもらう | UGC → 社会的証明 |
