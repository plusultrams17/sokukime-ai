## 1. 書籍/フレームワークベースの検証

### 1-1. リーンスタートアップ (Eric Ries) — 評価: B+

現状の「B2Bチームプラン課金完成を最優先」という判断は、Build→Build→Buildの罠。
- **正しい順序**: (1) 個人ユーザーでFree→有料転換ファネルを実証 → (2) 利用データ蓄積 → (3) B2B営業
- B2Bは「お問い合わせフォーム + 手動対応」で十分な段階

### 1-2. ブルーオーシャン戦略 (Kim & Mauborgne) — 評価: A-

「メソッドあり × 個人向け価格」のポジションは日本市場に実在しないブルーオーシャン。

| アクション | 内容 | 評価 |
|-----------|------|------|
| 除去 | 初期導入費、営業担当オンボーディング | OK |
| 削減 | UI装飾を最小限、機能3本集中 | OK |
| 増加 | メソッド体系性、スコア精度 | 要強化 |
| 創造 | 個人が月¥990-4,980で法人品質 | 最大の強み |

### 1-3. ランチェスター戦略 — 評価: C+

ターゲットが広すぎる。一人運営の弱者戦略として不十分。

**推奨セグメント**: **保険営業 × 入社1-2年目 × アポはとれるが成約できない**
- MDRT文化で自己投資意識が高い
- 月¥990-1,980は保険営業の自己投資予算として現実的
- チャネルもXに集中

### 1-4. Crossing the Chasm — 評価: B

**ホールプロダクトに不足:**
- 管理者ダッシュボード（チームスコア一覧）: 未実装
- 導入支援ドキュメント: 未作成
- 成功事例: 存在しない → **最初の1社は無料パイロット**

### 1-5. THE LEAN B2B — 評価: B-

**問題発見インタビュー未実施が致命的。** 保険代理店/不動産仲介のマネージャー5人にインタビューが急務。

### 1-6. プロダクトレッドグロース — 評価: A-

Free→Paid設計は良いが、Time to Valueを前倒しすべき（デモスコアプレビューなど）。
