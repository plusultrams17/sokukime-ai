## Part 1: SEO技術監査

### 1.1 検出された技術的SEO問題

#### 重大度：高

| # | 問題 | 影響 | 対応策 |
|---|------|------|--------|
| 1 | `/learn` ページにmetadataがない | クライアントコンポーネントのため`export const metadata`が使えず、title/descriptionが親layoutから継承されるだけ | `generateMetadata` を使うか、`learn/layout.tsx` にmetadataを追加 |
| 2 | LP3ページ（`/lp/roleplay`, `/lp/learn`, `/lp/worksheet`）がsitemapに含まれているがcanonicalが未設定 | 重複コンテンツのリスク。Googleがメインページとの区別ができない | LP→メインページへのcanonical設定、またはnoindex設定 |
| 3 | 多数のページにJSON-LDが欠落 | `/features`, `/about`, `/use-cases`, `/learn`, `/referral` にスキーマなし | 全ページにBreadcrumbList + ページ固有スキーマ追加 |
| 4 | `<link rel="icon">` のmetadataが未設定 | ブラウザタブにfaviconが表示されない可能性 | layout.tsxの`metadata.icons`に追加 |

#### 重大度：中

| # | 問題 | 影響 | 対応策 |
|---|------|------|--------|
| 5 | 個別ページのcanonical URLが未設定 | `/features`, `/about`, `/use-cases` 等にcanonicalなし | 各ページの`metadata.alternates.canonical`を設定 |
| 6 | BreadcrumbListが一部ページのみ | ホームページ（1階層）とブログ記事のみ | 全ページにBreadcrumbList追加 |
| 7 | sitemap.xmlに `/learn` が含まれていない | 22レッスンの個別ページもインデックスされない | sitemapにlearn関連ページを追加 |
| 8 | ブログ記事が3本のみ | トピカルオーソリティが弱い | 月2-4本ペースでの記事追加 |
| 9 | 業種別ページが存在しない | 「不動産 営業 ロープレ」等の業種KWを取れない | `/industry/[slug]` ページテンプレート作成 |

#### 重大度：低

| # | 問題 | 影響 | 対応策 |
|---|------|------|--------|
| 10 | next.config.tsにリダイレクト/リライトなし | 将来のURL変更時に404リスク | 必要に応じて追加 |
| 11 | 画像のalt属性が一部SVGで欠落 | アクセシビリティとSEOに軽微な影響 | `aria-hidden="true"` は装飾画像で適切、情報画像にはalt追加 |
| 12 | Organization schemaのlogoがfavicon.ico | 適切なロゴ画像の方がリッチリザルトに有利 | 正式なロゴ画像（200x200以上）をアップロード |

### 1.2 robots.txt 最適化

**現状の問題:**
- AI検索エンジンのクローラー（GPTBot, ChatGPT-User, PerplexityBot等）に対する明示的な許可ルールがない
- 一部のサイトはこれらをブロックしているため、明示的にAllowすることで差別化できる

**最適化後のrobots.ts:**

```typescript
// AI検索エンジンのクローラーを明示的に許可
rules: [
  {
    userAgent: "*",
    allow: "/",
    disallow: ["/api/", "/auth/", "/roleplay"],
  },
  {
    userAgent: "GPTBot",        // ChatGPTのクローラー
    allow: "/",
    disallow: ["/api/", "/auth/", "/roleplay"],
  },
  {
    userAgent: "ChatGPT-User",  // ChatGPTのブラウジング
    allow: "/",
    disallow: ["/api/", "/auth/", "/roleplay"],
  },
  {
    userAgent: "PerplexityBot",  // Perplexityのクローラー
    allow: "/",
    disallow: ["/api/", "/auth/", "/roleplay"],
  },
  {
    userAgent: "Google-Extended", // Google AI Overview用
    allow: "/",
  },
  {
    userAgent: "Amazonbot",      // Alexa AI
    allow: "/",
    disallow: ["/api/", "/auth/", "/roleplay"],
  },
],
```

### 1.3 sitemap.xml 最適化

**追加すべきページ:**
- `/learn` （学習コースTOP）— priority 0.7
- 個別レッスンページ（将来的に静的生成する場合）
- `/industry/[slug]` 業種別ページ（新規作成後）
- 新しいブログ記事

**改善点:**
- `lastModified` を `new Date()` ではなく実際の更新日に設定（毎回変わるとGoogleが信頼しない）
- LP3ページにnoindex設定するならsitemapから除外

### 1.4 Core Web Vitals 改善点

| 指標 | 現状評価 | 改善案 |
|------|---------|--------|
| LCP (Largest Contentful Paint) | 良好（テキストベース） | Hero画像がなくテキスト中心のため良好。SVGイラストをCSS化で更に改善可能 |
| FID/INP (Interaction to Next Paint) | 良好 | Supabase初期化がサーバーサイドで行われるため問題なし |
| CLS (Cumulative Layout Shift) | 要確認 | `StickyCTA`や`ScrollSlideIn`の出現でCLSが発生する可能性。fixed/stickyで制御済みなら問題なし |
| TBT (Total Blocking Time) | 要確認 | GoogleAnalytics, ScrollDepthTrackerの読み込みが影響する可能性。`next/script`のstrategy="afterInteractive"で遅延読み込み推奨 |

**具体的な改善:**
1. フォント読み込み最適化: `display: 'swap'` をGeistフォントに追加（現状はNext.jsのデフォルトで適切）
2. JSON-LDの`dangerouslySetInnerHTML`は軽量なので問題なし
3. SVGイラストがインラインで多いが、Server Componentなので初期HTMLに含まれるためLCPへの影響は軽微

### 1.5 内部リンク構造の改善案

                                    ↕
                              /learn

**具体的な改善:**

2. **featuresページから関連ブログ記事へリンク:** 「詳しくはブログ記事で解説」
