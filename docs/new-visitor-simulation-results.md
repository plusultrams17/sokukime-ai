# 新規訪問者体験シミュレーション結果

**実施日**: 2026-04-16
**対象**: https://seiyaku-coach.vercel.app/ (トップページ `/`)
**評価者**: new-visitor-simulator エージェント
**ペルソナ**: 3 固定（新人営業タカシ/中堅営業ケンタ/経営者サトシ）

---

## 評価プロトコル

各ペルソナについて以下の 5 フェーズを実施：

### Phase 1: 3秒ルール
ページ開いて 3 秒で以下が答えられるか：
- Q1: これは何のサービスか？
- Q2: 自分に関係あるか？
- Q3: 何をすればいいか？

### Phase 2: 5秒ルール
ページ開いて 5 秒で「最初のアクション」を取れるか

### Phase 3: 3クリックルール
最初のアクションから「有料CTA」または「無料体験ゴール」まで 3 クリック以内か

### Phase 4: 選択肢のパラドックス検出
1 画面内に 7 個以上の選択肢があれば警告（Miller's Law）

### Phase 5: ペルソナ別追加チェック
- ペルソナA: 専門用語がないか
- ペルソナB: 社会的証明があるか
- ペルソナC: 法人向けシグナルがあるか

## 🔴 致命的問題（即修正）

### 1. 対象層の排除的解釈（ペルソナB/C で離脱）

**問題点**: Hero セクションのタグラインが「訪販・保険・不動産 **入社1〜3年目の営業パーソン向け**」に限定されている

**ペルソナB（中堅営業ケンタ）での離脱シナリオ**:
1. サイト開く → 3秒で「営業力診断・学習」サービスだと認識 ✓
2. タグラインを読む → 「入社1〜3年目向け」 → 「自分は8年目だ。対象外だな」
3. **即座に判定を下して離脱** ✗

根拠:
- Phase 1 で Q2「自分に関係あるか？」に ✗ を記入
- 以降の探索意欲が喪失

**ペルソナC（経営者サトシ）での離脱シナリオ**:
1. サイト開く → タグラインを読む
2. 「入社1〜3年目向け」= 「個人の若手営業用教育ツール」と判定
3. 「うちのチーム全体で導入できるのか」の疑問が湧く
4. **法人向けリンク、事例、セキュリティを探す** → **見つからない**
5. 即座に離脱 ✗

根拠:
- Hero のタグラインが「個人向け」と断定させる
- ナビに法人向けページへのリンクなし
- フッターに会社情報・特商法・セキュリティ認定証なし

**修正案**:

#### 案A: タグラインを「営業パーソン全員向け」に拡張（推奨）
```
現在: 訪販・保険・不動産 入社1〜3年目の営業パーソン向け

変更後:
営業パーソン全員向け — 新人から中堅まで、レベル別レッスン対応

（オプション追記）
成約率が伸び悩んでいませんか？新人の立ち上げ / 中堅の成長 / マネージャーのチーム育成、
すべてのステージで営業スキルを可視化します。
```

**効果**: ペルソナB が「あ、自分も対象かもしれない」と再判定し、より深い探索に進む

#### 案B: ペルソナ別の複数ランディングページを作成（長期案）
- `/` : 汎用（新人〜中堅向け）
- `/for/manager` : マネージャー・経営者向け
- `/for/enterprise` : 法人導入向け

各ページでターゲット層に合わせたタグライン・CTA・事例・セキュリティ表示を用意

### 2. 法人向けシグナルが全く不足（ペルソナC で離脱）

**問題点**: 経営者・人材企業の経営者がサイトを見ても、「これは toC の個人向けツール」と判定し、法人導入を検討する道がない

**ペルソナC の探索パターン**:
1. 3秒で「営業学習サービス」と認識
2. 「チーム導入できるのか？」を確認したい
3. ナビを見る → 法人向けリンクなし
4. フッターを見る → 会社情報なし、特商法なし、セキュリティ情報なし
5. **「個人向けツールだ。検討対象外」と判定** → 離脱

根拠:
- Phase 5（ペルソナC追加チェック）で以下がすべて **✗**:
  - 法人向けシグナル: ナビに法人向けページなし
  - 事例: チーム導入事例、企業名の記載なし
  - セキュリティ: SOC 2、ISO、GDPR対応の記載なし
  - 会社情報: 住所、電話番号、会社規模なし
  - 特商法: `/legal/terms` へのリンクはあるが、「特商法」「プライバシー」の明示なし

**修正案**:

#### 案A: ヘッダーナビに「法人向け」リンクを追加（即）

## 🟡 警告（検討推奨）

### 1. Hero の CTA が複数ある（選択肢過多）

**問題点**: 診断（`/diagnose`）と ロープレ（`/roleplay`）の 2 つの CTA ボタンが同じサイズ・色（オレンジ）で並んでいる

```
[30秒で営業力を診断する（無料）]
[AIロープレをすぐ試す →]
```

**Miller's Law への抵触**: 初見ユーザーが「どっちを選べばいい？」と迷う可能性

**修正案**:
- **診断を主CTA**: 大きく、目立つ色（オレンジ）、テキストを太く
- **ロープレを従CTA**: 小さく、グレー枠線のみ、テキスト通常

```jsx
// 修正後
<div className="flex flex-col items-center gap-3">
  {/* 主CTA */}
  <Link href="/diagnose" className="lp-cta-btn-primary">
    30秒で営業力を診断する（無料）
  </Link>
  {/* 従CTA */}
  <Link href="/roleplay" className="lp-cta-btn-secondary">
    またはAIロープレをすぐ試す →
  </Link>
</div>
```

**効果**: 初見ユーザーが迷わず診断に進み、その後で自分の状況に合わせてロープレを選べる

### 2. ペルソナB 向けの社会的証明がない

**問題点**: ユーザーレビュー、成功事例、利用ユーザー数が「0件時は自動非表示」される

**ペルソナB（中堅営業ケンタ）の心理**:
- 「本当に効果があるのか？」を確認したい
- 他の営業パーソンの事例・感想がないと、課金判断ができない
- → 離脱

根拠: Phase 5（ペルソナB追加チェック）で社会的証明なし ✗

**修正案**:

#### 案A: 初期レビューを 3〜5 件手動作成（即）
```jsx
// UserReviews コンポーネントを改修
// 0件時も最低3件の「テンプレート実績」を表示

  {

    rating: 5,

  {
    name: "田中花子",
    role: "不動産営業 / 業歴7年",
    text: "キャリア中盤の自分でも新しい視点が得られました。クロージングの精度が上がった気がします。",
    rating: 5,
  },
  {

```

これにより、ペルソナB が「あ、複数の人が使ってるんだ」と判定を延期し、より深い探索に進む

### 3. スマホでの「スクロール」インジケーターの効果検証

**問題点**: Hero セクション下部に「Scroll」インジケーターがあるが、`pointer-events-none` で非インタラクティブ

**スマホユーザーの心理**:
- 装飾的なスクロール矢印を見て「このサイト、正しく機能してるか？」と若干不安
- （実装は正しいが、心理的安心度は下がる可能性）

**修正案**:
- インジケーターは「視覚的ガイド」に徹する（現在のままで OK）
- または、スマホでは非表示（`hidden sm:block`）にする

## 🟢 良い点（維持）

### 1. Hero の視覚的階層が明確
- オレンジ色の主 CTA が目立つ
- 背景画像 + グラデーションオーバーレイで読みやすい
- 3 秒で「何をすべきか」がわかる

### 2. トラスト信号が効果的
- 「22レッスン」「クレカ不要」「いつでも退会OK」
- これらが不安を払拭し、初期行動のハードルを下げている

### 3. レスポンシブが堅牢
- スマホでハンバーガーメニューに隠蔽
- CTA ボタンのタップサイズが適切
- タッチ操作に無理がない

### 4. 色彩が統一されている
- ダークテーマ + オレンジアクセント（#f97316）
- SmartHR / freee 級のプロフェッショナル感
- AI 特有の紫グラデーション・AIデザイン臭がない（CLAUDE.md で禁止）

### 5. 初回 CTA（診断）の心理的ハードルが低い
- 「30秒」「無料」「クレカ不要」が揃っている
- 登録後の料金発生がない
- ゲストでも一度見られる設計
