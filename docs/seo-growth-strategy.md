# 成約コーチ AI — オーガニック集客設計書

## 概要

無料ツール + 業種別ランディングページによるオーガニック集客の仕組み。
登録不要の無料ツールでSEO流入を獲得し、体験→登録→課金の導線を構築する。

---

## 1. 無料ツール戦略 (Free Tool Strategy)

### 目的
- SEO流入の入り口を増やす（検索キーワードの面を広げる）
- 「登録不要で使える」体験価値を提供
- 結果表示時にメインサービス（AIロープレ）への導線を設置

### ツール一覧

| ツール名 | URL | ターゲットKW | 種類 |
|---------|-----|-------------|------|
| 営業力診断テスト | `/tools/sales-quiz` | 営業力 診断, 営業スキル チェック | Quiz/Assessment |
| トークスクリプト生成 | `/tools/script-generator` | 営業 トークスクリプト, 営業 台本 | Generator |
| 反論切り返しトーク集 | `/tools/objection-handbook` | 営業 反論処理, 切り返しトーク | Reference/Library |
| クロージング率計算 | `/tools/closing-calculator` | 成約率 計算, 営業 KPI | Calculator |

### 各ツールの評価スコア (Free Tool Strategy Scorecard)

| 評価項目 | 診断テスト | スクリプト生成 | 反論トーク集 | 成約率計算 |
|---------|-----------|-------------|------------|-----------|
| 検索需要 | 4 | 5 | 5 | 3 |
| ターゲット適合度 | 5 | 5 | 5 | 4 |
| ユニーク性 | 4 | 3 | 4 | 3 |
| 製品への導線 | 5 | 5 | 4 | 4 |
| 実装容易性 | 4 | 4 | 5 | 5 |
| メンテ負荷(逆) | 5 | 4 | 5 | 5 |
| リンク獲得性 | 3 | 4 | 5 | 3 |
| シェア性 | 5 | 3 | 4 | 3 |
| **合計** | **35** | **33** | **37** | **30** |

全ツールが25点以上 = Strong candidate。

### 導線設計

```
Google検索
  ↓ (SEOで上位表示)
無料ツール使用（登録不要）
  ↓ (結果表示 + CTA)
AIロープレ体験（1回無料）
  ↓ (価値を実感)
無料アカウント登録
  ↓ (継続利用で価値を実感)
Proプラン課金
```

### 各ツールの実装仕様

#### 1. 営業力診断テスト (`/tools/sales-quiz`)
- **機能**: 10問の選択式クイズ → 5項目レーダー評価
- **技術**: クライアントサイドのみ（API不要）
- **SEO**: 300語以上の解説テキスト + SoftwareApplication JSON-LD
- **CTA**: 「弱点をAIロープレで克服する」

#### 2. トークスクリプト生成 (`/tools/script-generator`)
- **機能**: 業種×商材×ターゲット → 5ステップのスクリプト生成
- **技術**: テンプレートベース（AI API不使用）、クリップボードコピー機能
- **SEO**: スクリプトの書き方解説 + 5ステップメソッド説明
- **CTA**: 「生成したスクリプトでAIロープレしてみる」

#### 3. 反論切り返しトーク集 (`/tools/objection-handbook`)
- **機能**: 6カテゴリ×5パターン = 30の反論切り返しパターン
- **技術**: 静的コンテンツ + カテゴリフィルター
- **SEO**: FAQPage JSON-LD（30のQ&A）、超テキストリッチ
- **CTA**: 「反論処理をAIロープレで実践する」

#### 4. クロージング率計算ツール (`/tools/closing-calculator`)
- **機能**: KPI入力 → ファネル可視化 + 業種別ベンチマーク比較
- **技術**: クライアントサイド計算、リアルタイム更新
- **SEO**: 成約率の改善方法解説 + 業種別ベンチマーク表
- **CTA**: 「AIロープレで成約率を上げる」

## 2. 業種別ページ戦略 (Programmatic SEO)

### 目的
- 「[業種] 営業」「[業種] ロープレ」等のロングテールKWを網羅
- 各業種の営業パーソンに「自分向けのサービスだ」と感じてもらう
- 12業種 × 複数KW = 100+ のキーワードカバレッジ

### URL構造
```
/industry                    ← 業種一覧（Hub）
/industry/insurance          ← 保険営業
/industry/real-estate        ← 不動産営業
/industry/reform             ← リフォーム営業
/industry/exterior-painting  ← 外壁塗装営業
/industry/solar              ← 太陽光営業
/industry/automotive         ← 自動車営業
/industry/recruitment        ← 人材紹介営業
/industry/it-saas            ← IT/SaaS営業
/industry/advertising        ← 広告営業
/industry/medical-devices    ← 医療機器営業
/industry/printing           ← 印刷営業
/industry/bridal             ← ブライダル営業
```

### ターゲットキーワード一覧

| 業種 | 主要KW | ロングテールKW |
|------|-------|--------------|
| 保険 | 保険 営業 ロープレ | 保険 営業 コツ, 生命保険 クロージング |
| 不動産 | 不動産 営業 練習 | 不動産 営業 トーク, 住宅 営業 ロープレ |
| リフォーム | リフォーム 営業 | リフォーム 営業 トーク, リフォーム 反論処理 |
| 外壁塗装 | 外壁塗装 営業 | 外壁塗装 営業 コツ, 塗装 飛び込み営業 |
| 太陽光 | 太陽光 営業 | 太陽光パネル 営業 トーク, 太陽光 クロージング |
| 自動車 | 自動車 営業 | カーディーラー 営業 コツ, 車 営業 トーク |
| 人材紹介 | 人材紹介 営業 | 人材 営業 トーク, 求人 営業 ロープレ |

| 印刷 | 印刷 営業 | 印刷会社 営業 コツ, 印刷 新規開拓 |
| ブライダル | ブライダル 営業 | ウェディング 営業, ブライダル クロージング |

### ページ構成（テンプレート）

各業種ページは以下のセクションで構成:

1. **ヒーロー** — 業種名入りのh1 + サブタイトル + 業種統計
2. **課題セクション** — その業種特有の3つの課題（カード形式）
3. **反論パターン** — 4-5個のアコーディオン（客の反論 → 切り返し例）
4. **ロープレシナリオ** — 3つの練習シナリオ（難易度バッジ付き）
5. **SEOテキスト** — 200語以上の業種×AI営業の解説
6. **FAQ** — 4-5問（FAQPage JSON-LD付き）
7. **他の業種** — 11業種へのリンク（内部リンク網）
8. **CTA** — 業種名入りのアクションボタン

### 技術実装

- **データソース**: `src/data/industries.ts`
- **テンプレート**: `src/app/industry/[slug]/page.tsx`
- **静的生成**: `generateStaticParams()` で12業種を事前生成
- **SEO**: `generateMetadata()` で動的メタタグ
- **JSON-LD**: FAQPage + BreadcrumbList を自動生成

## 4. SEO技術仕様

### メタタグテンプレート
- **title**: `{ページ固有タイトル} | 成約コーチ AI`
- **description**: 120-160文字、主要KWを含む
- **canonical**: 正規URLを設定

### JSON-LD 構造化データ
- **ホーム**: SoftwareApplication + FAQPage + BreadcrumbList
- **業種ページ**: FAQPage + BreadcrumbList
- **ツールページ**: SoftwareApplication + FAQPage（該当する場合）

### サイトマップ
- Next.js の `app/sitemap.ts` で自動生成（既存の場合は更新）
- 全業種ページ + 全ツールページを含める

### ページ速度
- 全ツールはクライアントサイド完結（API呼び出しなし）
- 業種ページは静的生成（generateStaticParams）
- 画像なし（SVGアイコン + CSS）

## 6. KPI・効果測定

- 業種ページ → ロープレ遷移率
